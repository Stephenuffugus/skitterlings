THE ONE BLOCKER — postMessage protocol mismatch (must align before we repoint)
---------------------------------------------------------------------------
Earning will NOT work the moment we point at your live URL, because our host
earn-bridge and your game speak DIFFERENT postMessage protocols:

  - Your game emits:   { source:"skitterlings", type:"coins",
                         data:{ balance, delta, reason } }
  - Our host credits:  { source:"sunbeam-sdk", type:"earn", amount:N, src }
                       (validates origin -> calls our server earnHashes,
                        clamped <=200/msg, <=400/min)

Good news: your origin https://stephenuffugus.github.io is ALREADY in our host
allowlist (STUDIO_ORIGINS), so origin validation already passes. The issue is
purely the message shape — your game never sends the event our host listens for,
and it does not load our sunbeam-sdk.js.

PROPOSED FIX (entirely on OUR side, your game stays unchanged):

{ source:"skitterlings", type:"coins" } event and maps a positive `delta` to a
sunbeam earn (origin-validated + clamped, same as above).
